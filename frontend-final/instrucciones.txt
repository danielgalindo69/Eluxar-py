
Actúa como un desarrollador frontend con 40 años de experiencia en desarrollo de aplicaciones web de comercio electrónico. Tu tarea es completar el diseño y desarrollo del **frontend** de una plataforma llamada **Eluxar**, un marketplace especializado en la venta de perfumes para hombre y mujer que integra comercio electrónico, gestión administrativa e inteligencia artificial.

**Aclaración importante:** Tu rol es exclusivamente de **frontend**. No debes implementar lógica de backend, base de datos ni servicios externos. Todas las integraciones con API, autenticación, pagos e inteligencia artificial deben simularse con datos mock o estados locales mientras el backend no esté disponible. El frontend debe estar preparado para conectarse fácilmente al backend real cuando este esté listo.

**Stack tecnológico del frontend:**
- Framework: React (SPA)
- Estilos: el que consideres más adecuado según el proyecto existente
- Gestión de estado: el que consideres más adecuado (Context API, Redux, Zustand, etc.)
- Enrutamiento: React Router
- Comunicación con API: preparada para conectarse a un backend en Java Spring Boot mediante API REST en formato JSON sobre HTTPS
- Control de roles simulado: ADMIN, EMPLEADO, CLIENTE

**Páginas ya existentes en el proyecto:**
- catalog
- cart
- checkout
- auth
- admin
- admin/products
- admin/orders
- admin/users

**Instrucción importante:** Puedes modificar o eliminar cualquiera de las páginas existentes si lo consideras necesario para mantener coherencia, consistencia visual o mejorar la arquitectura general del proyecto. Tu criterio como experto tiene prioridad.

**Instrucción sobre navegación:** Revisa la barra de navegación actual del proyecto y agrégale las rutas y enlaces que consideres necesarios para garantizar una experiencia de usuario completa, fluida y coherente. Si hay páginas nuevas que deban ser accesibles desde el menú principal, el menú de usuario, el panel administrativo o el footer, agrégalas según tu criterio profesional. No te limites a las páginas listadas, si identificas rutas adicionales que mejoren la navegabilidad del sistema, créalas e incorpóralas.

**Páginas que debes desarrollar organizadas por módulo:**

**Módulo 1 – Gestión de Usuarios y Acceso**
- /register → Registro de nuevo usuario con validación de campos, email único y aceptación de términos
- /forgot-password → Recuperación de contraseña mediante token enviado al correo registrado
- /profile → Gestión de perfil del usuario: datos personales y contraseña
- /profile/addresses → Registro, edición y eliminación de múltiples direcciones de envío con opción de dirección predeterminada

**Módulo 2 – Catálogo y Productos**
- /product/:id → Ficha de detalle del producto con imágenes, descripción técnica, familia olfativa, variantes de volumen y stock disponible
- /search → Buscador de productos por nombre, marca o ingrediente con resultados en tiempo real
- /catalog (modificar existente) → Agregar sistema de filtros avanzados por precio, familia olfativa, marca y tamaño

**Módulo 3 – Inventario y Stock**
- /admin/inventory → Panel de movimientos de inventario: registro de entradas y salidas de almacén con historial auditado
- /admin/stock-alerts → Visualización de productos con stock crítico y configuración de umbrales mínimos de alerta
- (La reserva de stock y actualización post-pago deben prepararse como componentes listos para conectarse al backend cuando esté disponible)

**Módulo 4 – Ventas y Checkout**
- /order-confirmation → Pantalla de confirmación de pedido con resumen de compra, número de orden y mensaje de éxito tras pago exitoso
- /order-history → Historial de órdenes del cliente con estados y detalle de cada compra

**Módulo 5 – Pagos y Logística**
- /admin/payments → Panel de confirmaciones de pago, actualización de estado de órdenes simulada con datos mock
- /admin/shipping → Gestión de envíos: visualización de guías de despacho y seguimiento de estado por orden
- /order/:id/edit-address → Pantalla para modificar la dirección de entrega antes del despacho

**Módulo 6 – Inteligencia Artificial**
- /fragrance-test → Test olfativo interactivo con cuestionario de preferencias, resultados simulados con datos mock listos para conectar al motor de recomendación
- /recommendations → Pantalla de recomendaciones personalizadas con productos sugeridos basados en el perfil del usuario, simulada con datos mock
- /chat → Chatbot de asesoría con interfaz conversacional lista para conectarse al servicio de IA del backend

**Módulo 7 – Admin y Reportes**
- /admin/dashboard → Panel principal con KPIs: métricas de ventas, conversión, productos más vendidos y niveles de inventario, simulados con datos mock
- /admin/banners → Gestión de banners promocionales: creación, edición, activación/desactivación y orden de aparición en la página principal
- /admin/prices → Gestión de precios: actualización de precios por producto con validación de valor positivo y reflejo inmediato en catálogo
- /admin/categories → Gestión de categorías y marcas: creación, edición, eliminación con validación de productos asociados
- /admin/images → Gestión de imágenes por producto: carga en formato JPEG/PNG/WEBP con límite de 2MB y generación automática de miniaturas

---

**Requisitos generales que deben aplicarse a todas las páginas:**
- Diseño completamente responsive para móvil, tablet y escritorio
- Control de acceso simulado por roles en frontend: ADMIN y EMPLEADO acceden al panel administrativo, CLIENTE solo a las páginas públicas y de su cuenta
- Simulación de expiración de sesión tras 15 minutos de inactividad
- Todos los formularios deben validar datos en frontend antes de cualquier llamada a API
- Los mensajes de error deben ser claros y no exponer información técnica sensible
- Las acciones importantes (eliminar, guardar, cerrar sesión) deben mostrar mensajes de confirmación
- Todas las llamadas a API deben estar centralizadas y preparadas para apuntar a un backend real en el futuro
- El diseño debe ser coherente con las páginas ya existentes en el proyecto
